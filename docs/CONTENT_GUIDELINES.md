# Content Guidelines

## Purpose

These guidelines keep CMS content consistent across the Version 1.0 architecture portfolio.

Content should feel editorial, calm, and precise. It should describe the work clearly without turning the studio into a sales pitch.

## Naming Rules

- Use title case for project names.
- Keep names short, concrete, and architectural.
- Prefer names tied to place, material, light, or spatial type.
- Avoid campaign-style names, slogans, and overly abstract titles.

Good examples:

- Courtyard Residence
- Stone Apartment
- North Light Gallery
- Basalt Room

Avoid:

- The Ultimate Luxury Villa
- Modern Dream Space
- Experience Elegance

## Writing Tone

- Write in quiet editorial language.
- Describe space, atmosphere, material, light, and context.
- Use complete sentences.
- Avoid hype, urgency, superlatives, and marketing claims.
- Do not use lorem ipsum.

Preferred words:

- restrained
- measured
- quiet
- threshold
- courtyard
- light
- material
- proportion
- atmosphere

Avoid:

- world-class
- stunning
- cutting-edge
- game-changing
- bespoke luxury

## Category Taxonomy

Use the following Version 1.0 categories:

- Residential
- Interior
- Architecture
- Hospitality

Each project belongs to one primary category.

## Services Taxonomy

Use the existing CMS service values:

- Architecture
- Interior Design
- Master Planning
- Furniture Design
- Landscape

Assign only services that are clearly represented in the project.

## Location Format

Use separate fields for city and country.

Format:

```text
City
Country
```

Examples:

- Paros, Greece
- Milan, Italy
- Sonoma, United States

For homepage display, city may be used as the compact location and country as supporting context.

## Image Guidelines

- Use real project photography when available.
- Prefer large architectural images with clear spatial depth.
- Avoid cropped detail images as cover images.
- Avoid dark, heavily stylized, or decorative imagery.
- Cover images should work at wide desktop scale and stacked mobile scale.
- Gallery images should include at least three images per project.
- Temporary placeholder images must have clear alt text and should be replaced before production launch.

## Excerpt Rules

- Keep excerpts to one sentence.
- Target 120-180 characters.
- Describe the project calmly and specifically.
- Mention place, material, atmosphere, or spatial idea.
- Do not repeat the project title unless needed for clarity.

Example:

```text
A courtyard residence shaped by stone, shade, and open thresholds near the Aegean coast.
```

## Description Rules

- Use one short paragraph for Version 1.0 seed content.
- Describe the architectural idea, not the sales value.
- Mention spatial organization, material logic, or relationship to site.
- Keep descriptions suitable for future project detail pages.

## Publishing Rules

- Only publish content ready to appear publicly.
- Featured homepage projects must also be published.
- Keep `order` values intentional and stable.
- Use realistic years and consistent status values.
