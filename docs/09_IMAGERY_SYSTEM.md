# 09 — Imagery System

## Purpose

Photography is the foundation of the website.

Images are not decorative assets.

They are the primary content.

Every interface decision should support the presentation of architecture and interior photography.

---

# Philosophy

Architecture is experienced through space.

The website should communicate space through imagery.

Photography should create emotion before text explains anything.

The visitor should understand the quality of the studio before reading a single paragraph.

---

# Photography Principles

Every image should communicate one or more of these qualities:

* Space
* Light
* Material
* Texture
* Scale
* Atmosphere
* Geometry
* Craftsmanship

Images that do not contribute to these qualities should not be used.

---

# Image Quality

Only professional photography should be used.

Requirements:

* High resolution
* Natural lighting whenever possible
* Consistent color grading
* Sharp details
* Proper exposure
* No visible compression artifacts

Never use low-quality images.

---

# Color Consistency

All photography should feel like it belongs to the same studio.

Maintain consistency in:

* White balance
* Contrast
* Saturation
* Temperature
* Shadows
* Highlights

Avoid mixing drastically different editing styles.

---

# Image Priority

The importance of image types should follow this order:

1. Exterior Architecture

2. Interior Spaces

3. Material Details

4. Furniture & Objects

5. Human Presence (optional)

Architecture should always remain the main subject.

---

# Human Presence

People may appear in photography.

However:

Architecture remains the hero.

People should communicate scale and atmosphere.

Avoid posed commercial lifestyle photography.

Natural movement is preferred.

---

# Image Cropping

Respect the original composition.

Avoid aggressive cropping.

Never crop important architectural lines.

Allow the building to breathe.

---

# Hero Images

Hero photography should:

* create immediate emotional impact;
* establish atmosphere;
* introduce the project;
* communicate scale.

The hero image should work without additional visual effects.

---

# Gallery Principles

Project galleries should tell a story.

Recommended order:

1. Exterior Overview

↓

2. Main Interior

↓

3. Spatial Sequence

↓

4. Details

↓

5. Materials

↓

6. Closing Atmosphere

The gallery should feel cinematic rather than chronological.

---

# Image Sizes

Use a mix of:

* Fullscreen
* Full-bleed
* Large editorial images
* Vertical compositions
* Horizontal compositions

Avoid pages made only from identical image sizes.

Variation creates rhythm.

---

# Image Layout

Preferred layouts:

* Single large image
* Two-image composition
* Offset image
* Vertical storytelling gallery
* Editorial sequence

Avoid:

* Masonry galleries
* Dense thumbnail grids
* Pinterest-style layouts

The presentation should feel curated.

---

# Captions

Captions are optional.

If used, they should remain understated.

Typical information:

* Project name
* Photographer
* Material
* Space

Captions should never compete with the imagery.

---

# Photography and Typography

Text should never cover important architectural details.

When text overlays an image:

* ensure strong contrast;
* preserve readability;
* avoid visual clutter.

Whenever possible, separate text from photography instead of placing it on top.

---

# Motion with Images

Animation should reveal images.

Not distract from them.

Preferred effects:

* Clip-path reveal
* Fade with subtle movement
* Slow scale adjustment
* Gentle parallax
* Crossfade transitions

Avoid:

* Rotation
* Bounce
* Flip
* Zoom explosions
* Fast movement

---

# Image Loading

Performance is part of the visual experience.

Requirements:

* Lazy loading for non-critical images
* Priority loading for hero images
* Responsive image sizes
* AVIF/WebP where possible
* Smooth loading transitions

Images should appear gracefully without layout shifts.

---

# Accessibility

Every image must include:

* meaningful alt text;
* descriptive metadata where appropriate.

Decorative images should be marked appropriately.

---

# Future CMS Integration

Every project image should support:

* Alt text
* Caption
* Orientation
* Display order
* Photographer credit (optional)
* Featured flag

The CMS should manage content.

The frontend should define presentation.

---

# Final Principle

Photography is the architecture.

The interface is only the gallery.

If the visitor remembers the images long after leaving the website, the imagery system has succeeded.
