# 05 — Design Language

## Purpose

This document defines the visual language of the interface.

It is not about colors or components.

It is about how the interface communicates.

Every future design decision should be consistent with this language.

---

# Core Idea

The website should feel like architecture.

Not software.

Not marketing.

Not a landing page.

Every screen should resemble an architectural composition rather than a user interface.

---

# Interface Philosophy

The interface should almost disappear.

Visitors should remember:

the projects,

the spaces,

the atmosphere,

not the UI itself.

The interface is a frame.

Architecture is the artwork.

---

# Visual Communication

Communicate through:

* photography
* typography
* proportion
* composition
* rhythm
* whitespace

Never through visual noise.

---

# Visual Weight

Every element has visual weight.

Only important elements deserve heavy visual weight.

Highest weight:

* Hero photography
* Project photography
* Editorial headlines

Medium weight:

* Navigation
* Section titles
* Metadata

Lowest weight:

* Buttons
* Labels
* Small UI elements
* Supporting information

---

# Hierarchy

Users should immediately understand what matters.

Priority:

Architecture

↓

Images

↓

Titles

↓

Descriptions

↓

Metadata

↓

Navigation

↓

UI

---

# Rhythm

Pages should feel rhythmic.

Alternate:

Large

↓

Small

↓

Image

↓

Whitespace

↓

Typography

↓

Image

↓

Whitespace

Avoid repeating identical sections.

Avoid visual monotony.

---

# Scale

Think in architectural scale.

Some elements should feel monumental.

Others should feel intimate.

Hero headlines can be extremely large.

Metadata should remain almost invisible.

Contrast in scale creates elegance.

---

# Alignment

Alignment should feel intentional.

Avoid centering everything.

Prefer:

* left alignment
* asymmetric balance
* controlled offsets
* editorial layouts

---

# Negative Space

Empty space is content.

Whitespace is not decoration.

Whitespace controls:

* attention
* pacing
* elegance
* emotion

Never fill empty space because it feels empty.

---

# Image Behavior

Images should breathe.

Do not over-crop.

Allow architecture to exist inside the frame.

Prefer:

large images

few images

high quality images

over

many small thumbnails.

---

# Text Behavior

Text should be concise.

Every sentence should matter.

Avoid marketing language.

Avoid buzzwords.

Use calm editorial language.

---

# Interface Density

Low density.

Never crowd the interface.

Large spacing.

Few elements.

Clear hierarchy.

Quiet pages.

---

# Navigation Philosophy

Navigation should always feel secondary.

It helps users move.

It never competes with the work.

The navigation should disappear into the experience.

---

# Motion Language

Motion should resemble camera movement.

Not object animation.

Imagine:

walking,

looking,

discovering,

revealing.

Not bouncing.

Not flying.

Not spinning.

---

# Contrast

Contrast should come from:

scale,

space,

light,

typography,

photography.

Not from bright colors.

---

# Authenticity

Nothing should feel artificial.

Avoid visual tricks.

Avoid unnecessary effects.

Real architecture is already beautiful.

The interface should respect that.

---

# Consistency

Every page should feel like it belongs to the same architectural studio.

Even if layouts differ,

the language should remain identical.

---

# Timelessness

Design trends change.

Architecture lasts.

The interface should aim for longevity.

A visitor should still find the website elegant years from now.

---

# Final Principle

The interface should never ask for attention.

It should quietly guide visitors toward the architecture.

If users remember the buildings instead of the interface,

the design language has succeeded.
