# 00 — Project Principles

## Purpose

This document defines the non-negotiable principles of the project.

Every design decision, animation, component, page, and line of code must follow these principles.

If a future decision conflicts with this document, this document always wins.

---

# Vision

The goal is not to build a website.

The goal is to build a digital architectural experience.

The website should feel like entering a carefully designed building.

Every page should feel like another room.

Every transition should feel like walking through architecture.

---

# Primary Goal

The website exists to present architecture.

The interface exists only to support that goal.

The interface must never become more important than the projects.

---

# Less Is More

When choosing between two solutions:

Choose the simpler one.

When choosing between adding or removing:

Remove first.

Complexity should never exist without purpose.

---

# Every Element Must Have Purpose

Nothing should exist because it looks cool.

Every element must answer:

Why does this exist?

If there is no clear answer, remove it.

---

# Motion Has Meaning

Animation is communication.

Animation is not decoration.

Motion should:

* explain
* guide
* reveal
* connect

Motion should never exist only to impress.

---

# Architecture Comes First

Everything supports architecture.

Priority:

Architecture

↓

Photography

↓

Typography

↓

Space

↓

Motion

↓

Interface

↓

Technology

Technology is invisible.

Architecture is visible.

---

# Premium Means Restraint

Premium products do not shout.

Premium products feel confident.

Confidence is created by:

* whitespace
* rhythm
* proportion
* typography
* photography
* precision

Never by decoration.

---

# Editorial Thinking

Think like a magazine.

Not like a landing page.

Large images.

Short texts.

Strong composition.

Quiet typography.

Slow rhythm.

---

# Design Before Development

Never solve design problems with code.

Solve them through better design decisions.

Development exists to implement design.

Not invent it.

---

# Content Drives Layout

The content determines the layout.

The layout should never force the content.

Photography should always remain the hero.

---

# CMS Does Not Control Design

Payload manages content.

Payload never defines layout.

Payload never controls spacing.

Payload never controls animations.

Payload never controls typography.

The frontend owns the experience.

---

# Performance Is Part Of Design

A slow premium website is not premium.

Performance is a design feature.

Smoothness is more important than visual complexity.

---

# Accessibility Is Mandatory

Good design is accessible.

Accessibility is not optional.

Every interaction should remain usable with:

* keyboard
* screen readers
* reduced motion
* sufficient contrast

---

# Consistency Over Novelty

Do not reinvent every page.

Create a system.

Repeat patterns.

Users should learn the interface once.

---

# Long-Term Thinking

Every component should be reusable.

Every decision should scale.

Every pattern should support future pages.

Avoid one-off solutions.

---

# Simplicity Wins

If something can be removed without reducing quality,

remove it.

If something can be simplified,

simplify it.

---

# Final Principle

The website should feel timeless.

Not trendy.

Years from now, it should still feel elegant, calm, and relevant.

We are not designing for today's trend.

We are designing for lasting quality.
