# 07 — Typography

## Purpose

Typography is one of the primary visual elements of the website.

It should communicate confidence, calmness, precision, and timeless elegance.

Typography is not decoration.

It is architecture made readable.

---

# Typography Philosophy

Typography should feel editorial.

Not corporate.

Not trendy.

Not experimental.

The goal is to create hierarchy through scale, spacing, rhythm, and restraint.

---

# Typography Roles

The system is divided into four roles:

```text
Display

↓

Heading

↓

Body

↓

UI
```

Each role has a specific purpose.

---

# Display

Display typography is used only for major moments.

Examples:

* Homepage Hero
* Project Hero
* Studio Manifesto

Characteristics:

* large
* elegant
* spacious
* expressive
* cinematic

Display text should never be used for long paragraphs.

---

# Heading

Headings organize the content.

They should remain visually strong but calmer than Display typography.

Examples:

* Section titles
* Project titles
* Page titles

Characteristics:

* readable
* balanced
* confident

---

# Body

Body typography communicates information.

Characteristics:

* neutral
* highly readable
* generous line height
* moderate line length

Avoid dense text blocks.

---

# UI Typography

Used for:

* Navigation
* Buttons
* Metadata
* Labels
* Forms

Characteristics:

* simple
* clean
* unobtrusive

UI text should never compete with headlines.

---

# Reading Width

Maximum comfortable width:

```css
max-width: 680px;
```

Editorial statements may be wider.

Long paragraphs should never span the entire screen.

---

# Hierarchy

Hierarchy should be created primarily by:

1. Size
2. Weight
3. Whitespace
4. Position

Not by excessive color changes.

---

# Line Height

Display:

Tight but elegant.

Heading:

Comfortable.

Body:

Generous.

UI:

Compact.

The interface should never feel crowded.

---

# Letter Spacing

Large headlines may use slightly tighter tracking.

Small UI elements may use slightly increased tracking.

Avoid extreme tracking values.

---

# Paragraph Spacing

Separate ideas through whitespace.

Do not rely on empty sentences.

Every paragraph should communicate one idea.

---

# Editorial Rhythm

Alternate between:

Large statement

↓

Supporting paragraph

↓

Whitespace

↓

Image

↓

Caption

↓

Whitespace

↓

Next statement

This rhythm creates a calm reading experience.

---

# Text Alignment

Preferred:

* Left aligned
* Occasionally asymmetric

Avoid centering large amounts of text.

Centered typography should only be used when it creates a deliberate visual moment.

---

# Typography and Photography

Typography should never overpower photography.

Images remain the primary storytelling element.

Text supports the narrative.

---

# Numbers and Metadata

Metadata should be subtle.

Examples:

* Year
* Location
* Area
* Status

These elements should always feel secondary.

---

# Quotes and Manifestos

Important statements should breathe.

Use:

* generous spacing
* large type
* minimal surrounding elements

A manifesto should feel like a page from an architecture book.

---

# Responsive Typography

Typography should scale fluidly.

Use modern CSS functions such as:

```css
clamp()
```

Avoid multiple breakpoint-specific font sizes whenever possible.

---

# Accessibility

Typography must remain readable.

Guidelines:

* sufficient contrast
* scalable text
* accessible line height
* readable font size
* avoid extremely thin font weights for body text

---

# Future Font Selection

The exact font families will be selected later.

Requirements:

* Premium appearance
* Excellent readability
* Editorial character
* Full Latin support
* Full Cyrillic support
* Variable font preferred
* Optimized for web performance

The typography system should not depend on a single commercial font.

---

# Final Principle

Typography should feel intelligent, quiet, and timeless.

Readers should notice the ideas before they notice the font.
