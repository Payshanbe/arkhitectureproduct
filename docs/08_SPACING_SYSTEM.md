# 08 — Spacing System

## Purpose

Spacing is one of the defining characteristics of the website.

Whitespace is not empty space.

Whitespace creates rhythm, elegance, focus, and visual hierarchy.

The website should never feel compressed.

---

# Philosophy

Architecture is built from proportions.

The interface should follow the same principle.

Spacing should create calmness.

Not density.

Every distance should feel intentional.

---

# Spacing Scale

Use one consistent spacing scale throughout the project.

Recommended scale:

```txt
4
8
12
16
24
32
48
64
80
96
120
160
200
```

Never invent random values.

Every spacing decision should come from this system.

---

# Section Spacing

Sections define the rhythm of the page.

Recommended desktop spacing:

Small Sections

96px

↓

Standard Sections

120px

↓

Hero Sections

160–220px

↓

Major Visual Breaks

240px

---

# Component Spacing

Every component should have internal breathing room.

Cards should never feel crowded.

Buttons should never feel oversized.

Forms should feel calm.

Lists should feel organized.

---

# Typography Spacing

Space around text is more important than font size.

Always separate:

Headline

↓

Supporting Text

↓

Image

↓

Next Section

Avoid stacking multiple paragraphs without generous spacing.

---

# Grid Spacing

Columns should have consistent gaps.

Recommended:

Desktop

24–32px

Tablet

20–24px

Mobile

16px

---

# Image Spacing

Images should never touch each other unless intentionally creating a gallery.

Large images deserve generous surrounding whitespace.

Small supporting images can sit closer together.

---

# Card Spacing

Cards should breathe.

Content inside cards should have:

* generous padding
* balanced spacing
* strong hierarchy

Never compress content to fit more information.

---

# Button Spacing

Buttons should feel lightweight.

Avoid oversized CTA buttons.

Buttons should sit comfortably inside the surrounding whitespace.

---

# Navigation Spacing

Navigation should remain spacious.

Logo and menu should never touch viewport edges.

Menu items should have generous spacing between them.

The header should feel calm.

---

# Footer Spacing

The footer should not feel like an afterthought.

Give it enough vertical space to become a natural ending to the experience.

---

# Mobile Spacing

Do not simply reduce every spacing value.

Maintain breathing room.

Reduce only where necessary.

The mobile version should still feel premium.

---

# Visual Rhythm

Pages should alternate between:

Large whitespace

↓

Content

↓

Whitespace

↓

Photography

↓

Whitespace

↓

Typography

↓

Whitespace

This rhythm creates a gallery-like browsing experience.

---

# Density

Low density is preferred.

Show less.

Give each element room.

Avoid trying to fit more content above the fold.

---

# Consistency

Spacing should feel invisible.

Users should never consciously notice spacing.

They should only feel that the interface is calm and effortless.

---

# Final Principle

If a layout feels crowded,

add space before adding design.

Whitespace is one of the strongest visual tools in architecture and should be treated as a first-class design element.
