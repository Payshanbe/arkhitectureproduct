# 20 — Information Architecture

## Purpose

This document defines how users move through the website.

It focuses on navigation, content relationships, and the overall information flow.

The goal is to make the website intuitive, calm, and effortless to explore.

---

# Information Architecture Philosophy

The website should not feel like a collection of disconnected pages.

It should feel like one continuous architectural journey.

Every page should naturally lead to the next.

Users should always know where they are and what to explore next.

---

# Primary Navigation

The primary navigation consists of four core pages:

```text
Home
Work
Studio
Contact
```

These four items should always remain visible and easy to access.

Avoid adding unnecessary top-level navigation items.

---

# User Journey

The ideal visitor journey is:

```text
Landing
↓
Hero Experience
↓
Featured Projects
↓
Work
↓
Project Detail
↓
Studio
↓
Contact
```

Alternative journeys should also feel natural.

---

# Homepage Flow

```text
Hero
↓
Selected Projects
↓
Studio Introduction
↓
Featured Project
↓
Contact
```

Purpose:

Introduce the studio emotionally before asking visitors to explore further.

---

# Work Flow

```text
Work
↓
Project Category (optional)
↓
Project Card
↓
Project Detail
↓
Next Project
```

Visitors should never reach a dead end.

---

# Project Detail Flow

Every project page should encourage further exploration.

Recommended flow:

```text
Hero
↓
Project Story
↓
Gallery
↓
Project Metadata
↓
Next Project
```

The "Next Project" section is essential to keep users engaged.

---

# Studio Flow

```text
Manifesto
↓
Studio Story
↓
Approach
↓
Team
↓
Contact
```

The Studio page builds trust rather than selling services.

---

# Contact Flow

The Contact page should remove friction.

Content order:

```text
Editorial Statement
↓
Email
↓
Phone
↓
Address
↓
Social Links
↓
Contact Form (optional)
```

---

# Navigation Principles

Navigation should always answer three questions:

* Where am I?
* Where can I go next?
* How do I return?

The navigation should never become the visual focus.

---

# Breadcrumbs

Breadcrumbs are not required.

The website structure is intentionally shallow.

Users should navigate through:

* Header
* Next Project
* Work page

instead of breadcrumb trails.

---

# Footer Navigation

The footer acts as a secondary navigation.

Recommended links:

```text
Home
Work
Studio
Contact
Privacy Policy
```

---

# Project Discovery

Projects should be discoverable through:

* Featured Projects
* Work Page
* Next Project
* Category Filters

Users should never rely on search in version 1.

---

# Search

Search is not part of the initial release.

It can be added later if the project portfolio grows significantly.

---

# Categories

Categories help organize projects without overwhelming users.

Suggested categories:

```text
Residential
Commercial
Hospitality
Interior Design
Architecture
```

Keep the number of categories small.

---

# Dead Ends

Avoid dead-end pages.

Every page should provide a clear next step.

Examples:

* Next Project
* Back to Work
* Contact
* Studio

---

# URL Structure

Recommended structure:

```text
/
/work
/work/[slug]
/studio
/contact
```

URLs should remain clean, readable, and descriptive.

---

# Internal Linking

Every project page should link to:

* another project;
* the Work page;
* the Contact page.

This improves both navigation and SEO.

---

# Mobile Navigation

On mobile:

* Use a fullscreen menu.
* Keep navigation depth to one level.
* Avoid nested menus.

Navigation should remain simple and touch-friendly.

---

# Final Principle

Visitors should always feel invited to continue exploring.

No page should feel like the end of the experience.
